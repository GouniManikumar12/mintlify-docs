# JSON Schemas

All AIP payloads conform to these JSON schemas.
Each file defines required fields, validation rules, and example payloads.

## Available Schemas

Schemas are organized into **Advertising** (platform → operator → brand agents → auction) and **PAG** (Permissioned Access / Publisher: platform ↔ publisher retrieval, no ads).

### Advertising

| Schema | Description |
|--------|--------------|
| [Platform Request](/schemas/platform-request) | Platform → Operator: user context, intent, runtime, auction params |
| [Context Request](/schemas/context-request) | Operator → Brand Agents: enriched auction context |
| [Agentic Intent Taxonomy](/schemas/agentic-intent-taxonomy) | Operator-neutral decision envelope for conversational intent classification and monetization gating |
| [Bid](/schemas/bid) | Brand Agent bid payload (pricing, recommendation) |
| [Auction Result](/schemas/auction-result) | Operator → Platform: winning response |
| [event-cpx.json](event-cpx.json) | Exposure event |
| [event-cpc.json](event-cpc.json) | Click event |
| [event-cpa.json](event-cpa.json) | Conversion event |
| [ledger-record.json](ledger-record.json) | Final ledger record summary |

### PAG (Publisher / Permissioned Access)

| Schema | Description |
|--------|--------------|
| [aip.json — Publisher Declaration](/schemas/aip) | `/.well-known/aip.json` (AIP v0.1): consent, editorial_domains, retrieval limits |
| [RetrieveRequest](/schemas/retrieve-request) | Platform → publisher: permissioned content retrieval (no ads, no bidding) |
| [RetrieveResponse](/schemas/retrieve-response) | Publisher → platform: chunks, citations, or denial |
| [Access & Citation Events](/schemas/access-citation-event) | Post-retrieval accountability: access and citation events (no ads, no pricing, no user identity) |

## Usage

These schemas are provided in JSON Schema Draft 2020-12 format. You can use them to:

- **Validate** your payloads before sending to AIP endpoints
- **Generate** client code using tools like `quicktype` or `json-schema-to-typescript`
- **Test** your integration with sample data
- **Reference** exact field names, types, and constraints

## Integration Examples

### Validating a Context Request (Node.js)

```javascript
const Ajv = require('ajv');
const contextRequestSchema = require('./context-request.json');

const ajv = new Ajv();
const validate = ajv.compile(contextRequestSchema);

const payload = {
  context_id: "ctx_92fA1",
  session_id: "sess_001",
  operator_id: "admesh_operator",
  platform_id: "openai_chat",
  query_text: "best CRM for small teams",
  locale: "en-US",
  geo: "US",
  intent: {
    type: "commercial",
    decision_phase: "compare",
    context_summary: "User evaluating CRM tools",
    turn_index: 3
  },
  allowed_formats: ["weave", "citation"],
  timestamp: new Date().toISOString(),
  auth: {
    nonce: "nonce_123",
    sig: "sig_hmac_123abc"
  }
};

if (validate(payload)) {
  console.log('Valid payload');
} else {
  console.error('Validation errors:', validate.errors);
}
```

### Generating TypeScript Types

```bash
npx json-schema-to-typescript schemas/context-request.json > types/ContextRequest.ts
```
