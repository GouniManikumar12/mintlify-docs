# Raw JSON Schema Files

All AIP payloads conform to these JSON schemas.
Each file defines required fields, validation rules, and example payloads.

## Available Schemas

| Schema | Description |
|---------|--------------|
| [Platform Request](/schemas/platform-request) | Platform → Operator request envelope for AI intent monetization |
| [Context Request](/schemas/context-request) | Operator → Brand Agents participation context (JSON schema) |
| [Agentic Intent Taxonomy](/schemas/agentic-intent-taxonomy) | Conversation-intent classification envelope (JSON schema) |
| [Bid](/schemas/bid) | Brand agent response payload (JSON schema) |
| [Platform Response](/schemas/auction-result) | Operator → Platform auction outcome (JSON schema) |
| event-cpx.json (schemas-json) | `exposure_shown` event schema |
| event-cpc.json (schemas-json) | `interaction_started` event schema |
| event-delegation-started.json (schemas-json) | `delegation_started` event schema |
| event-delegation-activity.json (schemas-json) | `delegation_activity` event schema |
| event-delegation-expired.json (schemas-json) | `delegation_expired` event schema |
| event-cpa.json (schemas-json) | `task_completed` event schema |
| ledger-record.json (schemas-json) | Final settlement record summary |

## Usage

These schemas are provided in JSON Schema Draft 2020-12 format. You can use them to:

- **Validate** your payloads before sending to AIP endpoints
- **Generate** client code using tools like `quicktype` or `json-schema-to-typescript`
- **Test** your integration with sample data
- **Reference** exact field names, types, and constraints

## Integration Examples

### Validating a Context Request (Node.js)

```javascript
const Ajv = require('ajv');
const contextRequestSchema = require('./context-request.json');

const ajv = new Ajv();
const validate = ajv.compile(contextRequestSchema);

const payload = {
  spec_version: "1.0",
  context_id: "ctx_92fA1",
  source_request_id: "req_92fA1",
  timestamp: new Date().toISOString(),
  operator: { operator_id: "admesh_operator" },
  platform: {
    platform_id: "openai_chat",
    software: { name: "chatgpt", version: "4.3-mini" }
  },
  session: { id: "sess_001", turn_index: 3 },
  surface: {
    channel: "conversation",
    interaction_mode: "text",
    platform: "web",
    locale: "en-US",
    country: "US"
  },
  intent: {
    type: "commercial",
    decision_phase: "consideration",
    confidence: 0.87,
    summary: "User evaluating CRM tools"
  },
  allowed_formats: ["weave", "tail", "product_card"]
};

if (validate(payload)) {
  console.log('Valid payload');
} else {
  console.error('Validation errors:', validate.errors);
}
```

### Generating TypeScript Types

```bash
npx json-schema-to-typescript schemas/context-request.json > types/ContextRequest.ts
```
