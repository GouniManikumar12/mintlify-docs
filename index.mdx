### What is AIP?

AIP is an open protocol that governs how commercial entities participate in AI-driven decisions  -  from intent detection to selection, delegation, and outcome-based settlement.

**Short version:** AIP defines how intent becomes action in AI systems.

AIP is designed to complement existing standards while addressing gaps specific to AI-native interfaces and conversational systems. PlatformRequest uses the AIP canonical request envelope to ensure interoperability across compliant systems.

---

### Why AIP exists

AI systems are becoming decision engines, not just information tools.

But there is no shared standard for:

- when commercial participation is appropriate
- which entities are allowed to participate
- how decisions transition into actions
- how outcomes are verified and settled

Today, platforms handle this inconsistently and opaquely.

**AIP introduces a common protocol for:**

- permissioned participation based on user intent
- fair selection of eligible commercial agents
- seamless delegation from AI to brand agents
- verifiable outcomes and transparent settlement

In simple terms:
**AIP ensures that commercial participation in AI systems is intentional, controlled, and accountable.**

---

### What AIP standardizes

| Layer | What it does | Why it matters |
|-------|-------------|----------------|
| **Context Ingestion** | Captures structured, privacy-safe intent | Enables precise understanding |
| **Participation Governance** | Determines if/when brands can participate | Prevents spam, preserves trust |
| **Real-Time Selection** | Selects eligible agents (auction or rules) | Ensures relevance, not just highest bid |
| **Interaction Mode** | Decides recommend vs delegate | Bridges decision to action |
| **Delegation Protocol** | Manages session handoff to agents | Enables real execution |
| **Event Verification** | Tracks full lifecycle events | Prevents fraud |
| **Settlement** | Resolves payment based on outcomes | Aligns incentives |

---

### Built-in guarantees

- One verified charge per lifecycle event
- Only the highest-value event in a lifecycle is billable
- Signed and auditable ledger
- Works with any compliant operator
- Privacy-safe by design  -  no personal identifiers required

### Example 1: Recommend Mode

1. User asks: **"Best accounting software for freelancers"**
2. Intent classified: commercial, consideration phase
3. Participation allowed  -  selection happens
4. AI integrates recommendation into its response
5. No session transfer  -  user continues in the AI
6. AIP verifies the participation event and settles

---

### Example 2: Delegate Mode

1. User says: **"Sign me up for QuickBooks"**
2. Intent classified: transactional, action phase
3. Delegation allowed  -  agent selected
4. AI asks user for confirmation
5. Session starts  -  user completes signup via brand agent
6. AIP verifies the outcome and settles CPA
